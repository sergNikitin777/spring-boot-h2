Разрабатываем систему работы с информацией о клиентах. Сущности
1. Клиент (client), атрибуты:
  1.1. Имя (name) - строка
  1.2. коллекция номеров мобильных телефонов (phone)
  1.3. идентификаторы если необходимы
2. Номер мобильного телефона (строка) - нормализованный номер в международном формате (вида +79139199192)
Требуется написать REST-сервис для ведения базы клиентов
 - получение списка клиентов,
 - добавление клиента
 - просмотр информации о клиенте (имя, список телефонов)
 - добавление, изменение номера телефона клиента
При сохранении номера телефона сервис должен валидировать корректность номера и
приводить его к международному формату, т.к. на вход поступает "сырой" ввод пользователя в произвольном формате,
например ("8-913-919-91-92" или например "+41 44 668 18 00")
Сервис предпочтительно реализовать в виде standalone приложения, для хранения данных использовать embedded db,
например h2, или можно хранить данные просто в чем-то типа HashMap.
Необходимо предоставить инструкции/скрипты для сборки/запуска приложения, проводить исследования "как его запустить"
мы не хотим.
